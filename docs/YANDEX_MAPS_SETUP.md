# Подключение Яндекс.Карт для поиска адресов в такси

## Почему это важно

Сейчас поиск адресов работает через бесплатный OpenStreetMap (Nominatim) — он не требует
регистрации, но данные по малым городам ЯНАО у него неполные (нет уровня «подъезд»,
иногда путает похожие микрорайоны). Яндекс.Карты в разы точнее знают российские адреса,
особенно в небольших северных городах.

## Как включить

1. Зайти на **https://developer.tech.yandex.ru/** и войти через Яндекс ID.
2. Создать новый проект (кнопка «Подключить API»).
3. Подключить **два** сервиса в этом проекте:
   - **API Геосаджеста** (Geosuggest API) — для подсказок при вводе
   - **HTTP Геокодер** — для определения адреса по геолокации («📍 Моё местоположение»)
4. Скопировать выданный ключ (одинаковый для обоих сервисов, если они в одном проекте).
5. Открыть `js/app.js`, найти строку:
   ```js
   const YANDEX_MAPS_API_KEY="";
   ```
   Вставить ключ между кавычками:
   ```js
   const YANDEX_MAPS_API_KEY="ваш-ключ-здесь";
   ```
6. Сохранить файл — поиск адресов такси автоматически переключится на Яндекс.Карты.
   Ничего больше менять не нужно.

## Что если ключ невалиден или лимит исчерпан

Приложение это не заметит критично — при любой ошибке запроса к Яндексу автоматически
происходит откат на прежнюю систему (OpenStreetMap), пользователь просто получит чуть
менее точные подсказки, но поиск не сломается.

## Лимиты бесплатного тарифа Яндекса

На момент подключения уточните текущие условия в личном кабинете разработчика —
Яндекс периодически меняет бесплатные квоты. Для центра с несколькими сотнями заявок
такси в месяц обычно достаточно базового бесплатного тарифа, но при росте нагрузки
может понадобиться платный план.
