# Гармония — помощник ЦСОН (ЯНАО)

Веб-приложение в виде чат-бота для ГБУ ЯНАО «ЦСОН Гармония»: каталог
социальных услуг с ценами и скидкой «Морошка», корзина, запись к
специалистам, контакты, FAQ, экстренная помощь и личный кабинет.

## Структура проекта

```
garmoniya/
├─ index.html          разметка (подключает css + js)
├─ css/
│  └─ styles.css       все стили
├─ js/
│  ├─ data.js          справочные данные (услуги, цены, сотрудники, FAQ, города)
│  ├─ db.js            слой работы с базой данных (Supabase) + откат на localStorage
│  └─ app.js           логика приложения
├─ data/               редактируемые копии данных в JSON (services / staff / faq)
└─ README.md
```

## Как запустить локально

`js/db.js` подгружает Supabase через `import(...)`, поэтому открывать
нужно через локальный сервер, а не двойным кликом (`file://`).

```bash
# из папки garmoniya/
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

Без настроенной БД приложение работает в локальном режиме: данные
хранятся в браузере (localStorage), ничего настраивать не нужно.

## Как менять услуги, цены и сотрудников

Сейчас данные живут в `js/data.js`. Папка `data/*.json` — это удобные
для чтения копии тех же данных (например, чтобы посмотреть/согласовать
прайс). Меняешь `js/data.js` — обновляется приложение.

## Подключение базы данных (Supabase)

1. Зарегистрируйся на https://supabase.com и создай проект.
2. В **Settings → API** скопируй `Project URL` и `anon public` ключ.
3. Вставь их в `js/db.js` в объект `CONFIG`.
4. В **SQL Editor** выполни:

```sql
create table orders (
  id          bigint generated always as identity primary key,
  client_name text,
  client_phone text,
  city        text,
  moroshka    boolean,
  total       integer,
  items       jsonb,
  created_at  timestamptz default now()
);

create table bookings (
  id          bigint generated always as identity primary key,
  ticket      text,
  client_name text,
  client_phone text,
  city        text,
  department  text,
  specialist  text,
  visit_date  date,
  visit_time  text,
  comment     text,
  created_at  timestamptz default now()
);

-- Разрешаем вставку с публичного ключа (чтение закрыто)
alter table orders   enable row level security;
alter table bookings enable row level security;
create policy "anon insert orders"   on orders   for insert to anon with check (true);
create policy "anon insert bookings" on bookings for insert to anon with check (true);
```

После этого заявки и записи будут сохраняться на сервере.
Если БД временно недоступна, приложение не падает — данные остаются
в localStorage.

## ⚠️ Персональные данные и закон (ФЗ-152)

Приложение собирает ФИО, телефон и **СНИЛС**. ЦСОН — государственное
учреждение, поэтому действует ФЗ-152:

- Персональные данные граждан РФ должны храниться в БД **на территории РФ**.
- Supabase/Firebase по умолчанию хранят данные за рубежом — это подходит
  для прототипа, но **не для боевого запуска** с реальными СНИЛС.
- Для продакшена используйте российский хостинг БД (Yandex Cloud,
  VK Cloud или собственный Supabase на VPS в РФ). В коде поменяется
  только `SUPABASE_URL`.
- Рекомендуется не хранить СНИЛС в localStorage в открытом виде.

## Новые функции (обновление)

- **Сворачиваемое меню** — секции в главном меню разворачиваются/сворачиваются по нажатию на заголовок.
- **Помощник-маршрутизатор** (🤖) — понимает вопросы на естественном языке (синонимы, формы слов, мелкие опечатки) и даёт кнопку-переход в нужный раздел. Работает офлайн, без ключей и платных API. Логика и список «намерений» — в `js/features.js` (`ASST_INTENTS`), легко дополнять.
- **Калькулятор права на бесплатное обслуживание** (🧮) — короткий опрос с предварительной оценкой. Окончательное решение остаётся за специалистом (это указано в интерфейсе).
- **Новости и анонсы** (📰) — список в `js/data.js` (`newsData`) и копия в `data/news.json`. Чтобы обновить новости, редактируйте `newsData`.

### Почему помощник — не «настоящая» нейросеть
Полноценную языковую модель нельзя встраивать в статический сайт: её API-ключ попал бы в публичный код на GitHub, и им мог бы воспользоваться кто угодно за ваш счёт. Поэтому помощник работает как умный поиск по разделам. Если позже появится бэкенд (этап 5), его можно заменить на обращение к модели через серверный прокси.

## Вход через Госуслуги (ЕСИА) — что нужно для боевого режима

Сейчас кнопка «Войти через Госуслуги» — заглушка (`esiaLogin()` в `js/features.js`).
Чтобы она заработала по-настоящему, требуется:

1. **Регистрация информационной системы** в ЕСИА через Минцифры (Госуслуги для организаций). Физлицу/без официального статуса ИС подключиться нельзя.
2. **Серверная часть (бэкенд).** ЕСИА работает по протоколу OAuth 2.0 / OpenID Connect, и `client_secret` обязан храниться на сервере — в коде статического сайта его держать нельзя.
3. Поток входа: кнопка → редирект на `https://esia.gosuslugi.ru/...` → пользователь авторизуется → ЕСИА возвращает код на ваш бэкенд → бэкенд обменивает код на данные пользователя (ФИО, СНИЛС подтверждены) → создаёт сессию.

Большой плюс: после подключения ЕСИА **ручной ввод СНИЛС больше не нужен** — данные приходят подтверждёнными, и снимается часть рисков по ФЗ-152.

Несколько профилей получателей + редактирование своих данных; мультиязычность (перевод всех текстов интерфейса).

#### ⚠️ Очень важно про админку
1. **Пароль виден в коде.** На статическом сайте пароль `300897` доступен любому, кто откроет исходник. Это не защита, а мягкий замок. Рекомендуется как минимум сменить пароль в `js/admin.js` (`ADMIN`), но по-настоящему безопасным вход станет только с бэкендом (Supabase Auth).
2. **Правки локальны.** Изменения сохраняются только в браузере администратора (localStorage) и не видны другим.
3. **Как сделать правки постоянными:** в админке нажмите «Экспорт JSON» — скачается `garmoniya_data.json`. Передайте его разработчику: содержимое вставляется в `branchContent`/`cityData`, и правки становятся частью проекта для всех. Либо повторно загрузите файл через «Импорт JSON» на другом устройстве.
4. С бэкендом (этап 5) всё это заменяется на защищённый вход и общую базу, где правки сразу видят все.
