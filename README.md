ДОКУМЕНТАЦИЯ
Веб-приложение «Гармония — помощник »
Виртуальный помощник для граждан: прейскурант социальных услуг, запись к специалисту,
калькулятор льгот, личный кабинет, админпанель и мультиязычный интерфейс






Разработчик:
Туманов Илья Александрович

1. Описание процесса разработки
1.1. Предпосылки и постановка задачи
В рамках работы филиала ГБУ ЯНАО «ЦСОН Гармония» в г. Губкинский была выявлена потребность в цифровом инструменте, который бы упростил взаимодействие граждан с центром социального обслуживания. Основная аудитория центра — граждане пожилого возраста, люди с инвалидностью и семьи с детьми .
Поставил себе задачу: разработать веб-приложение в формате виртуального помощника (чат-бота), которое обеспечит удобный доступ к прейскуранту, записи к специалистам, справочной информации и личному кабинету получателя услуг. Приложение должно быть доступно через интернет, работать на любом устройстве (телефон, планшет, компьютер) и не требовать установки.
1.2. Проектирование архитектуры
На этапе проектирования я принял решение реализовать приложение как статический веб-сайт , что обеспечивает:
•	Мгновенную загрузку 
•	Работоспособность при нестабильном интернете
•	Простоту развёртывания и обновления
Данные (услуги, цены, сотрудники, контакты) хранятся в структурированном виде непосредственно в JavaScript-файлах. Это позволяет легко редактировать информацию, а также даёт возможность наполнять филиалы данными через встроенную админпанель.
Архитектура проекта была выстроена модульно — каждый функциональный блок вынесен в отдельный файл:
Файл	Ответственность
data.js	Справочные данные всех филиалов: услуги, сотрудники, контакты, новости, мероприятия, FAQ
app.js	Ядро приложения: навигация, корзина, запись, личный кабинет, избранное, поиск
features.js	Умный помощник, калькулятор льгот, обратный звонок, соцработник на дом, мероприятия
extras.js	Доступность (шрифт, тема, языки), профили, календарь, статистика, оценка, поздравления, онбординг, связь с оператором
admin.js	Админпанель: редактирование данных филиалов, экспорт/импорт JSON
db.js	Слой сохранения данных с возможностью расширения
1.3. Этапы разработки
Разработка велась поэтапно. На каждом этапе приложение оставалось работоспособным — новые функции добавлялись без нарушения существующих.

Этап 1. Базовая структура и прейскурант
Создан интерфейс чат-бота с выбором филиала, прейскурантом услуг из 19 категорий, корзиной заявок и отправкой на email центра. Реализован поиск услуг по названию, переключатель скидки «Морошка» и экран авторизации с вводом ФИО, телефона и СНИЛС.

Этап 2. Запись к специалисту и личный кабинет
Добавлена форма записи с выбором отделения, специалиста, даты и времени. Реализован личный кабинет с историей заявок, записей, избранными услугами и разделом «Мои документы». Внедрены статусы заявок, повтор заявки из истории и отмена записи.

Этап 3. Умный помощник и дополнительные сценарии
Разработан виртуальный помощник-маршрутизатор, понимающий вопросы на естественном языке. Добавлены сценарии: обратный звонок, вызов соцработника на дом, каталог мероприятий с записью, калькулятор права на бесплатное обслуживание.

Этап 4. Данные по филиалам и админпанель
Перестроена архитектура данных: каждый филиал получил индивидуальные услуги, сотрудников и контакты. Создана админпанель для редактирования данных филиалов через интерфейс бота. Внесён полный список сотрудников филиала г. Губкинский (57 человек, 10 отделений) на основании штатной расстановки.

Этап 5. Доступность и мультиязычность
Реализованы: регулятор размера шрифта (5 ступеней), тёмная тема, мультиязычный интерфейс (русский, английский). Добавлены профили получателей для оформления заявок за родственников.

Этап 6. Обратная связь и вовлечение
Внедрены: система оценки качества обслуживания (1–5 звёзд с комментарием), сезонные поздравления с 8 государственными и профессиональными праздниками, анимированный онбординг-тур для новых пользователей, мультиканальная связь с оператором через мессенджер Макс (max.ru), телефон и email.

Этап 7. Интеграция брендинга и полировка
Интегрированы официальные логотипы: «Морошка» (Единая карта жителя ЯНАО), Госуслуги (ЕСИА). Выполнен редизайн карточек меню с уникальными градиентами и анимацией. Устранены ошибки масштабирования и горизонтальной прокрутки. Оптимизирован интерфейс для мобильных устройств.
1.4. Используемые технологии
Технология	Применение

CSS3	Стилизация, анимации, тёмная тема, адаптивность, CSS-переменные
JavaScript (ES6+)	Логика приложения, работа с DOM, localStorage, генерация .ics и .txt файлов

localStorage	Хранение пользовательских данных, настроек, истории заявок

1.5. Подход к дизайну
Дизайн интерфейса создавался с учётом основной аудитории — людей старшего возраста и людей с ограниченными возможностями. Ключевые принципы:
•	Крупные элементы управления (минимум 44×44 пикселя для тач-устройств)
•	Контрастные цвета и читаемые шрифты с возможностью увеличения
•	Минимальное количество шагов для основных действий
•	Подсказки и обучение при первом входе
•	Поддержка тёмной темы для комфорта в вечернее время
•	Формат чат-бота — привычный и интуитивно понятный для всех возрастных групп
1.6. Тестирование
Тестирование проводилось на каждом этапе разработки:
•	Синтаксическая проверка всех JavaScript-файлов (автоматическая)
•	Функциональное тестирование: навигация, корзина, запись, профили, избранное, админпанель
•	Тестирование умного помощника на наборе из 20+ типовых запросов
•	Проверка сезонных поздравлений на все 8 праздничных дат
•	Тестирование на мобильных устройствах (горизонтальная прокрутка, масштабирование, тач-элементы)
•	Проверка тёмной темы для всех элементов интерфейса
•	Тестирование переключения языков и профилей получателей
1.7. Перспективы развития
Приложение спроектировано с заделом на расширение. Запланированные направления:
•	Подключение серверной части для онлайн-хранения заявок и записей
•	Интеграция с ЕСИА (Госуслуги) для подтверждённой авторизации
•	Реальные свободные слоты записи (без двойных записей)
•	SMS- и push-уведомления о статусе заявки
•	Установка как мобильное приложение (PWA) 
•	Озвучка интерфейса для слабовидящих пользователей

2. О проекте
Веб-приложение «Гармония — помощник» — интерактивный виртуальный помощник для граждан, обращающихся за социальными услугами в ГБУ ЯНАО «ЦСОН «Гармония».
Приложение реализовано в формате чат-бота и обеспечивает удобный доступ к прейскуранту, записи к специалистам, справочной информации и личному кабинету получателя услуг.
2.1. Поддерживаемые филиалы
•	г. Губкинский (полностью заполнен: 19 категорий услуг, 57 сотрудников)
•	г. Муравленко (заготовка для заполнения)
•	г. Ноябрьск (заготовка для заполнения)
•	г. Тарко-Сале (заготовка для заполнения)
•	пгт. Уренгой (заготовка для заполнения)
У каждого филиала — индивидуальные данные: услуги, цены, сотрудники, контактная информация. При переключении города интерфейс автоматически отображает данные выбранного филиала.

3. Функциональные возможности
3.1. Услуги и заявки
•	Прейскурант: 19 категорий социальных услуг с ценами (обычными и по карте «Морошка»)
•	Поиск услуг по названию
•	Корзина с подсчётом суммы и отображением экономии по карте «Морошка»
•	Отправка заявки на услуги по email 
•	Повторная заявка из истории одним нажатием
•	Избранные услуги (★) с быстрым добавлением в корзину
3.2. Запись к специалисту
•	Выбор отделения, специалиста, даты и времени
•	Автоматическая генерация номера талона
•	Экспорт записи в календарь телефона (.ics файл с напоминанием за 1 час)
•	Отмена записи из личного кабинета с уведомлением на email центра
•	Статусы заявок: «Принята» / «В работе» / «Выполнена»
3.3. Дополнительные сценарии
•	Обратный звонок — заявка с выбором удобного времени
•	Вызов социального работника на дом — форма с адресом и описанием
•	Каталог мероприятий с онлайн-записью на участие
•	Калькулятор права на бесплатное обслуживание 
3.4. Умный помощник
Виртуальный помощник понимает вопросы на естественном языке и направляет в нужный раздел. Поддерживает приветствия и светскую беседу, обращается по имени-отчеству, отвечает на частые вопросы из базы FAQ.
3.5. Связь с оператором
Мультиканальная связь с оператором центра: телефон, мессенджер Макс , электронная почта.
3.6. Личный кабинет
•	История заявок со статусами и функцией повтора
•	История записей с отменой и экспортом в календарь
•	Избранные услуги
•	Раздел «Мои документы»: заявление, согласие ФЗ-152, памятка «Морошка», права получателя, бланк жалобы
3.7. Профили получателей
Возможность добавить нескольких получателей услуг (Мама, Папа, Бабушка и др.) и переключаться между ними.
3.8. Оценка качества
После отправки заявки пользователю предлагается оценить удобство (1–5 звёзд) и оставить комментарий.
3.9. Сезонные поздравления
Бот автоматически поздравляет с праздниками: Новый год, 23 февраля, 8 Марта, День Победы, День защиты детей, День соцработника, День пожилого человека, Международный день инвалидов.
3.10. Обучение при первом входе
Анимированный онбординг-тур из 8 шагов знакомит нового пользователя с возможностями приложения.

4. Доступность и интерфейс
4.1. Панель доступности
•	Регулятор размера шрифта (А− / А+) — 5 ступеней, от 13px до 20px
•	Тёмная тема (переключатель) — полноценная тёмная тема для всех элементов
•	Выбор языка интерфейса: русский, English.
4.2. Адаптивность
Интерфейс оптимизирован для мобильных устройств. Масштабирование заблокировано. Горизонтальная прокрутка отключена. Меню прокручивается горизонтально для экономии пространства.
6. Админпанель
6.1. Вход
5 быстрых нажатий по логотипу в шапке.
6.2. Возможности
•	Выбор филиала для редактирования
•	Контакты: адрес, телефоны, email, часы работы
•	Услуги: категории, цены, добавление и удаление
•	Сотрудники: ФИО, отделение, должность, контакты
•	Новости и мероприятия: добавление, редактирование, удаление
•	Экспорт / Импорт JSON, Сброс
7. Памятка для администратора
1.	Откройте бот.
2.	Сделайте 5 быстрых нажатий по логотипу в шапке.
3.	Введите логин и пароль.
4.	Выберите филиал.
5.	Редактируйте: Контакты / Услуги / Сотрудники / Новости.
6.	Нажмите «Сохранить».
8. Вход через Госуслуги (ЕСИА)
Кнопка с официальным логотипом присутствует на экране авторизации. Для полноценного подключения требуется регистрация информационной системы в ЕСИА через Минцифры и серверная часть для хранения секрета OAuth.
