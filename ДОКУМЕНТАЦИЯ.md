# Документация и инструкция — «Гармония» (помощник ЦСОН ЯНАО)

Версия документа: для сборки v3 · Дата: июнь 2026

Это полное руководство по проекту: что это, как устроено, как запустить,
опубликовать, редактировать данные, пользоваться админпанелью и подключить
базу данных. В конце — краткая памятка для администратора.

## 1. О проекте

«Гармония» — веб-приложение в виде чат-бота для ГБУ ЯНАО «ЦСОН Гармония».
Помогает жителям:

- посмотреть прейскурант социальных услуг с ценами (в т.ч. со скидкой по карте «Морошка»);
- собрать корзину услуг и отправить заявку;
- записаться к специалисту, заказать обратный звонок или соцработника на дом;
- проверить право на бесплатное обслуживание;
- узнать контакты, новости и мероприятия;
- получить экстренную психологическую помощь;
- вести личный кабинет (заявки, записи, избранное).

Приложение поддерживает **5 филиалов**: Губкинский, Муравленко, Ноябрьск,
Тарко-Сале, Уренгой. У каждого филиала свои услуги, сотрудники и контакты.

Сейчас это **статический сайт** (работает без сервера). Заявки уходят на
почту центра через почтовый клиент. Для постоянного хранения заявок,
защищённого входа администратора и онлайн-записи предусмотрено подключение
базы данных (раздел 9).

## 2. Возможности

- Прейскурант: 19 категорий услуг (у Губкинского), цены обычные и по «Морошке».
- Поиск услуг по названию.
- Корзина с подсчётом суммы и экономии по «Морошке».
- Заявка на услуги, запись к специалисту, обратный звонок, соцработник на дом.

- Калькулятор права на бесплатное обслуживание.
- Умный помощник: понимает вопросы своими словами, приветствия, обращается по имени, отвечает по FAQ.
- Личный кабинет: заявки (со статусами), записи (с отменой), избранные услуги, повтор заявки.
- Новости, контакты, сотрудники, FAQ, экстренная помощь.
- Админпанель: редактирование услуг, цен, сотрудников и контактов по филиалам.

Все заявки и записи отправляются на **cson-gub@yanao.ru**.

```
garmoniya/
├─ index.html              разметка, подключает стили и скрипты
├─ css/
│  └─ styles.css           все стили
├─ js/
│  ├─ data.js              данные: услуги, сотрудники, контакты, новости, мероприятия, FAQ
│  ├─ db.js                слой базы данных (Supabase) + откат на localStorage
│  ├─ features.js          помощник, калькулятор льгот, новости, мероприятия, звонок, соцработник
│  ├─ admin.js             админпанель (вход, редактор, экспорт/импорт)
│  └─ app.js               основная логика (меню, корзина, запись, кабинет, избранное)
├─ data/                   редактируемые копии данных в JSON (для удобства)
│  ├─ services.json        услуги Губкинского (эталон)
│  ├─ staff.json           сотрудники Губкинского
│  ├─ news.json, events.json, faq.json
│  └─ branches/            заготовки филиалов (_TEMPLATE.json + пустые файлы)
├─ README.md               краткое описание
└─ ДОКУМЕНТАЦИЯ.md         этот файл
```

**Порядок подключения скриптов важен** (он задан в `index.html`):
`data.js → db.js → features.js → admin.js → app.js`.
Скрипты обычные (не модули), поэтому данные из `data.js` доступны во всех
остальных файлах как глобальные переменные.

## 5. Публикация на GitHub Pages

### Способ A — без командной строки (проще)

1. Зайдите на github.com → **New repository**. Имя, например, `garmoniya`, тип **Public**, **Create repository**.
2. На странице пустого репозитория нажмите ссылку **uploading an existing file**.
3. Перетащите **содержимое** папки `garmoniya` (файлы `index.html`, папки `css`, `js`, `data`), не саму папку.
4. Нажмите **Commit changes**.
5. **Settings → Pages → Source**: ветка `main`, папка `/ (root)` → **Save**.
6. Через минуту появится адрес вида `https://ВАШ_ЛОГИН.github.io/garmoniya/`.

### Способ B — через git

```bash
cd путь/к/garmoniya
git init
git add .
git commit -m "Гармония: первая публикация"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/garmoniya.git
git push -u origin main
```

Затем включите Pages, как в шаге 5 способа A.

### Обновление сайта после правок
Способ A: загрузите изменённые файлы заново (Add file → Upload files).
Способ B: `git add . && git commit -m "правки" && git push`.

## 6. Редактирование данных

Все справочные данные — в `js/data.js`. После правок сохраните файл и
обновите его на GitHub (раздел 5). Цены, услуги, сотрудников и контакты можно
менять также через админпанель (раздел 7).

### 6.1. Услуги и цены
Каждая категория — объект в массиве услуг филиала:

```js
{
  id: 1,
  name: "Чистка и уборка жилых помещений",
  icon: "🧹",
  rating: 4.9,
  items: [
    { n: "Мытьё окна (1 окно)", p: 300, m: 285 },  // p — обычная цена, m — по «Морошке»
    { n: "Мытьё полов",          p: 180, m: 171 }
  ]
}
```

- `p` — обычная цена в рублях.
- `m` — цена со скидкой «Морошка». Поставьте `null`, если скидки нет.
- `icon` — эмодзи категории.

### 6.2. Сотрудники
```js
{ dept: "Администрация", name: "Иванов Иван Иванович", pos: "Директор", ext: "701", email: "mail@yanao.ru" }
```
`dept` — отделение (по нему группируется список), `ext` — добавочный номер.

### 6.3. Контакты филиала
В объекте `cityData` в `js/data.js`:
```js
gubkin: {
  address: "629830, ЯНАО, г. Губкинский, ...",
  phone: "8(34936)2-70-77",   // как показывать
  phoneRaw: "73493627077",     // только цифры, для кнопки звонка
  email: "cson-gub@yanao.ru",
  hours: "Пн–Пт: 08:30–18:00",
  openH: 8, openM: 30, closeH: 18, closeM: 0  // для индикатора «открыто/закрыто»
}
```

### 6.4. Новости, мероприятия, FAQ
- Новости — массив `newsData` (`date`, `tag`, `title`, `text`).
- Мероприятия — массив `eventsData` (`id`, `date`, `title`, `place`, `desc`, `seats`).
- Частые вопросы — массив `faqData` (`q`, `a`).

### 6.5. Данные по филиалам
Контент каждого филиала хранится в `branchContent` (`js/data.js`):

```js
const branchContent = {
  gubkin:     { services: GUBKIN_SERVICES, staff: GUBKIN_STAFF }, // заполнен
  muravlenko: { services: [], staff: [] },  // пустая заготовка
  noyabrsk:   { services: [], staff: [] },
  tarko:      { services: [], staff: [] },
  urengoy:    { services: [], staff: [] }
};
```

Чтобы наполнить филиал: добавьте элементы в его массивы `services` и `staff`
по образцу Губкинского. Образец структуры — в `data/branches/_TEMPLATE.json`.
Если у филиала пусто, бот вежливо сообщит, что раздел в этом городе недоступен.
Контакты филиала меняются в `cityData` (см. 6.3).

### Возможности
1. Выбрать филиал для редактирования.
2. **Контакты** — адрес, телефоны, email, часы работы.
3. **Услуги** — категории и услуги: название, иконка, цены `p` и `m`; добавление и удаление категорий и услуг.

5. **Экспорт JSON** — выгрузить все правки в файл.
6. **Импорт JSON** — загрузить ранее выгруженный файл.
7. **Сброс** — вернуть исходные данные.

### ⚠️ Важно про админку (обязательно к прочтению)

1. **Пароль виден в коде.** На статическом сайте пароль `300897` доступен
   любому, кто откроет исходный код страницы. Это не настоящая защита, а
   «мягкий замок» от случайного входа. Минимум — смените пароль в `js/admin.js`
   (объект `ADMIN`). По-настоящему безопасным вход станет только после
   подключения бэкенда с защищённой авторизацией (раздел 9).

2. **Правки сохраняются только в браузере администратора** (в localStorage) и
   не видны другим пользователям сайта.

3. **Как сделать правки постоянными для всех:** в админпанели нажмите
   «Экспорт JSON» — скачается файл `garmoniya_data.json`. Передайте его
   разработчику: содержимое вставляется в `branchContent` и `cityData` в
   `js/data.js`, и правки становятся частью проекта. На другом устройстве тот
   же файл можно подгрузить через «Импорт JSON».

4. С базой данных (раздел 9) эти ограничения снимаются: появляется защищённый
   вход и общая база, где правки сразу видят все.

### Как сменить пароль/логин администратора
В файле `js/admin.js` в начале:

Поменяйте значения и обновите файл на GitHub. (Помните: на статике это всё
равно видно в коде — смена снижает риск, но не защищает полностью.)

## 8. Умный помощник

Помощник (кнопка «🤖 Спросить помощника») понимает вопросы на естественном
языке: подбирает нужный раздел, отвечает на приветствия и светскую беседу,
обращается к пользователю по имени-отчеству и отвечает на частые вопросы.

Это **не нейросеть**, а умный поиск по ключевым словам. Так сделано осознанно:
ключ настоящей языковой модели нельзя хранить в коде статического сайта (он
утёк бы из публичного репозитория). После подключения бэкенда помощника можно
заменить на обращение к модели через защищённый серверный прокси.

### Как добавить новый вопрос-ответ
В `js/features.js` в массиве `ASST_INTENTS`:
```js
{
  kw: ["ключевое", "слово", "синоним"],   // ищутся как подстроки (ловят формы слов)
  answer: "Текст ответа.",
  actions: [ { label: "Кнопка", fn: "showServices", cl: "teal" } ]  // fn — имя функции раздела
}
```
- `kw` — список ключевых слов (можно корни: «убор» поймает «уборка», «убирать»).
- `actions` — кнопки-переходы; `fn` — имя глобальной функции (`showServices`,
  `showBooking`, `showCategory` с `arg: номер` и т.д.).

Светская беседа и приветствия настраиваются в функции `smallTalk` там же.

## 9. Подключение базы данных (Supabase) — этап 5

Это превращает прототип в рабочую систему: заявки и записи сохраняются на
сервере, статусы можно вести, появляется основа для защищённой админки.

### Шаги
1. Зарегистрируйтесь на https://supabase.com, создайте проект.
2. **Settings → API**: скопируйте `Project URL` и `anon public` ключ.
3. Вставьте их в `js/db.js` в объект `CONFIG`:
   ```js
   const CONFIG = {
     SUPABASE_URL: "https://ВАШ_ПРОЕКТ.supabase.co",
     SUPABASE_ANON_KEY: "ВАШ_КЛЮЧ"
   };
   ```
4. В **SQL Editor** выполните:

```sql
create table orders (
  id           bigint generated always as identity primary key,
  client_name  text,
  client_phone text,
  city         text,
  moroshka     boolean,
  total        integer,
  items        jsonb,
  created_at   timestamptz default now()
);

create table bookings (
  id           bigint generated always as identity primary key,
  ticket       text,
  client_name  text,
  client_phone text,
  city         text,
  department   text,
  specialist   text,
  visit_date   date,
  visit_time   text,
  comment      text,
  created_at   timestamptz default now()
);

alter table orders   enable row level security;
alter table bookings enable row level security;
create policy "anon insert orders"   on orders   for insert to anon with check (true);
create policy "anon insert bookings" on bookings for insert to anon with check (true);
```

После этого заявки и записи начнут сохраняться на сервере. Если БД временно
недоступна, приложение не падает — данные остаются в localStorage.

### Что «оживёт» после подключения базы
- Реальные свободные слоты записи (без двойных записей).
- Загрузка фото и документов к заявке.
- Автоматические статусы заявок («Принята / В работе / Выполнена»).
- Аналитика для центра (какие услуги ищут чаще).
- Защищённый вход администратора и **общая** админка (правки видят все).

## 10. Вход через Госуслуги (ЕСИА)

Сейчас кнопка «Войти через Госуслуги» — заглушка. Чтобы она заработала:

1. Зарегистрировать информационную систему в ЕСИА через Минцифры (Госуслуги для организаций).
2. Поднять серверную часть: ЕСИА работает по OAuth 2.0 / OpenID Connect, и `client_secret` обязан храниться на сервере.
3. Поток: кнопка → редирект на ЕСИА → авторизация пользователя → возврат кода на ваш сервер → обмен кода на подтверждённые данные (ФИО, СНИЛС) → сессия.

Плюс: после ЕСИА ручной ввод СНИЛС не нужен — данные приходят подтверждёнными.

## 11. Персональные данные и ФЗ-152

Приложение собирает ФИО, телефон и СНИЛС. ЦСОН — государственное учреждение,
поэтому действует ФЗ-152 «О персональных данных»:

- Персональные данные граждан РФ должны храниться в базах **на территории РФ**.
- Supabase и Firebase по умолчанию хранят данные за рубежом — это подходит для
  прототипа, но **не для боевого запуска** с реальными СНИЛС.
- Для продакшена используйте российский хостинг БД (Yandex Cloud, VK Cloud или
  собственный сервер в РФ). В коде поменяется только `SUPABASE_URL`.
- Рекомендуется не хранить СНИЛС в localStorage в открытом виде; лучший вариант —
  подтверждённый вход через ЕСИА (раздел 10).

## 13. Решение проблем

- **Не загружаются шрифты / интерфейс «прыгает».** Google Fonts иногда медленно
  открывается в РФ. Решение — положить шрифты локально (папка `fonts/`) и
  подключить через `@font-face`.
- **Заявка не отправляется.** Кнопка открывает почтовый клиент; нужно нажать
  «Отправить» в нём. На устройствах без настроенной почты это может не работать —
  тогда поможет переход на бэкенд (раздел 9).
- **Изменения админки пропали.** Они хранятся в браузере; при очистке данных
  браузера/смене устройства теряются. Делайте «Экспорт JSON».
- **Supabase не подключается локально.** Открывайте сайт через локальный сервер
  (`python3 -m http.server`), а не двойным кликом.
- **Пустой раздел в каком-то городе.** У филиала ещё не заполнены услуги или
  сотрудники (раздел 6.5).

## 14. Краткая памятка для администратора

1. Откройте сайт центра.
2. На экране входа нажмите «🔐 Вход для администратора» (или зайдите в «Кабинет» → «Админпанель»).
3. Введите логин и пароль (выдаёт ответственный за сайт).
4. Вверху выберите **филиал**, который редактируете.
5. Переключайте вкладки:
   - **Контакты** — телефоны, email, адрес, часы; нажмите «Сохранить контакты».
   - **Услуги** — меняйте названия и цены, добавляйте/удаляйте; нажмите «Сохранить услуги».
   - **Сотрудники** — добавляйте и редактируйте; нажмите «Сохранить сотрудников».
6. Чтобы изменения увидели **все** пользователи, нажмите **«Экспорт JSON»** и
   передайте скачанный файл `garmoniya_data.json` разработчику сайта.
7. Кнопка **«Сброс»** возвращает исходные данные.

> Важно: пока сайт работает без сервера, ваши правки сохраняются только на этом
> компьютере. Поэтому шаг 6 (экспорт и передача файла) — обязательный, если нужно,
> чтобы изменения появились у всех. После подключения базы данных этот шаг
> перестанет быть нужным.

## 16. Обновление v5 — доступность, профили, тема, языки

### Панель доступности (под шапкой)
- **А− / А+** — регулятор размера шрифта (5 ступеней: 13–20px). Запоминается.
- **🌙 / ☀️** — переключатель тёмной темы. Полная тёмная тема для всех элементов.
- **Язык** — переключатель: русский, English, ненэцяˮ (ненецкий), хӑнты (хантыйский). Русский и английский заполнены полностью. Ненецкий и хантыйский — заготовки основных фраз (приветствия, меню), дополняет носитель языка.

### Несколько профилей получателей
Карточка «👨‍👩‍👦 Профили» в разделе «Кабинет». Позволяет добавить получателей (Мама, Папа, Бабушка и др.) с ФИО, телефоном и СНИЛС, и переключаться между ними. При оформлении заявки используются данные активного профиля.

### Экспорт записи в календарь (.ics)
- После оформления записи появляется кнопка «📅 Добавить в календарь».
- В кабинете у каждой записи кнопка «📅 В календарь».
- Скачивается .ics файл с напоминанием за 1 час.

### Статистика корзины
Карточка «📊 Статистика» в разделе «Кабинет». Показывает рейтинг популярных услуг с визуальными полосками. Данные собираются локально.

### Как добавить/дополнить переводы
В файле `js/extras.js` объект `I18N` содержит переводы по ключам. Для добавления языка или дополнения существующего: добавьте ключи по образцу русского (`ru`). Ключи, отсутствующие в выбранном языке, автоматически подставляются из русского.
